# What this is about?
I made this project while studying the [Crafting Interpreters](https://craftinginterpreters.com/) book, by Robert Nystrom.

Here, I wrote not only the code provided by the book, but also the proposed challenges and many personal notes/insights I had while developing the interpreter and exploring the uncharted lands my curiosity drove me to.

Since a significant part of the interpreter development is made in C, it was also a perfect opportunity to strengthen my knowledge in this fascinating language.

# A bit of context
While in my college course of System Analysis and Development, I had few opportunities do study and learn more about C programming, code optimization and low level development, since this course is more focused on the preparation for the job market and professional development.

But, somehow, the few classes I had in these subjects sparked me a fierceful curiosity, that droves me more and more towards this "deep dark side" of technology. 

After getting my degree, I started to look out on the internet to see if I could find somewhere, in the technology field, where all this concepts blend together in a meaningful way.

This search ended when I discovered about compilers and interpreters. An odd union of "deep into the machine" knowledge, that I've been so eager to work with, plus one of my most toiful hobbies: learning new languages.

# The Study Journal
While studying this book and doing it's challenges, I've felt the need to write down what I was learning. It was such a huge amount of information, a whole new world, that I would surely forget some things. 

My starving curiosity often lead me to ask silly questions that turned out to had fascinating answers. Thinking that would be such a loss to not register these too, gave me one more reason to jot them down.

Since this is a **learning journal**, you may notice some coding errors or a misunderstood concept here. I intend to, as soon as I get a solution to an error or understand that I wrote a mistaken info, come back and redo things properly. 

Yet, if you notice anything too grievous to be ignored, you can contact me email `ahrgo.prog@gmail.com`. I'll be pleased by the opportunity to learn how to improve these notes.

But hey, don't get it too serious. Sometimes I will write a totally awful code just to see what happens. It's fun to try out new things, and since this isn't production code, I can be a little more adventurous, I guess.
